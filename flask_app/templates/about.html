About the project

Process Pipeline:

  • Pull data from the API.
  • Store the data in a MongoDB database.
  • Clean the data to prepare it for the model.
  • Send the data through the prediction model using RandomForest.
  • Return probability of fraud and display in browser via Flask App

Final Model Performance:

•Accuracy: 97%
•Precision: 99%
•Recall: 97%
•F1 score: 98%

We tested performance on sample data using RandomForest, GradientBoosting, and a TensorFlow Keras Sequential model. The default RandomForest model performed the best, with no additional tuning of hyperparameters from default settings.

Next steps:

Provide a more interactive user interface where an e-commerce agent could view a list of all currently flagged transactions, see the predicted probability of fraud for each, and drill down to see additional details for a selected transaction.
